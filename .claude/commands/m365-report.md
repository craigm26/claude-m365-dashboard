Generate a Microsoft 365 tenant health report by querying the Microsoft Graph API via the `msgraph` MCP server.

## Instructions

Query each of the following sections using the Microsoft Graph MCP server. For each section, call the listed Graph API endpoints. If an endpoint fails or returns an error, note it as "unavailable" in that section and continue with the remaining sections.

After collecting all data, write the report to `reports/` with the filename format `YYYY-MM-DD_HHmmss_m365-health.md` and also display it in the conversation.

## Sections to Query

### 1. SharePoint Health
Call these endpoints:
- `GET /sites?$select=id,displayName,webUrl,lastModifiedDateTime` — list sites
- `GET /admin/sharepoint/settings` — tenant sharing/storage settings

Collect: total site count, recently modified vs stale (no activity in 90+ days), storage usage if available, any issues or warnings.

### 2. OneDrive Health
Call these endpoints:
- `GET /reports/getOneDriveUsageAccountDetail(period='D30')` — usage by account
- `GET /users?$select=id,displayName,assignedLicenses,signInActivity&$top=50` — user activity

Collect: total users, active vs inactive users (no sign-in in 30+ days), sync issues if reported, inactive licenses (users with licenses but no recent sign-in).

### 3. Copilot Adoption
Call these endpoints:
- `GET /reports/getMicrosoft365CopilotUsageUserDetail(period='D30')` — Copilot usage if available
- `GET /reports/getOffice365ActiveUserDetail(period='D30')` — general app usage as fallback

Collect: total Copilot licenses, active Copilot users, adoption rate, usage by app if available.

### 4. Permissions & Sharing
Call these endpoints:
- `GET /groups?$filter=groupTypes/any(c:c eq 'Unified')&$select=id,displayName,visibility&$top=100` — M365 groups
- `GET /admin/sharepoint/settings` — external sharing policy (reuse from SharePoint if already fetched)

Collect: public vs private groups count, external sharing policy status, any risky configurations.

## Error Handling

- If the MCP server is not connected, tell the user to authenticate first and provide setup instructions.
- If specific endpoints return 403/401, mark that section as "unavailable — insufficient permissions" and list the required scope.
- Always produce a report even if some sections fail — partial data is better than no report.
